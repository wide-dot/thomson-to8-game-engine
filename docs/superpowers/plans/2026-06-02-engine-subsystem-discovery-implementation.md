- [ ] **Step 2: Write catalog generation script skeleton**

Create `scripts/generate-engine-catalog.sh`:

```bash
#!/bin/bash
# Generate engine subsystem catalog from engine/ structure and usage analysis

OUTPUT_FILE="doc/engine-subsystems.md"

# Start markdown
cat > "$OUTPUT_FILE" << 'EOF'
# Thomson TO8 Engine Subsystems

Auto-generated catalog of all engine subsystems.
Last updated: $(date)

EOF

# Process each subsystem directory
for subsys in engine/*/; do
    subsys_name=$(basename "$subsys")
    
    # Extract purpose from any README or comments in the subsystem
    # For now, placeholder — we'll fill in by hand or from code comments
    
    echo "## $subsys_name (engine/$subsys_name/)" >> "$OUTPUT_FILE"
    echo "" >> "$OUTPUT_FILE"
    echo "**Purpose:** [to be filled from code analysis]" >> "$OUTPUT_FILE"
    echo "" >> "$OUTPUT_FILE"
    echo "**Used in:** [detecting from game-projects...]" >> "$OUTPUT_FILE"
    echo "" >> "$OUTPUT_FILE"
    echo "**Core Files:**" >> "$OUTPUT_FILE"
    find "$subsys" -name "*.asm" | sed 's/^/- /' >> "$OUTPUT_FILE"
    echo "" >> "$OUTPUT_FILE"
    
done

echo "Catalog generated: $OUTPUT_FILE"
```

Make executable:
```bash
chmod +x scripts/generate-engine-catalog.sh
```

- [ ] **Step 3: Write usage detection (grep for includes)**

Update `scripts/generate-engine-catalog.sh` to detect which games use each subsystem.

Insert this after the "Purpose" line:

```bash
    # Find which games include this subsystem
    used_in=""
    for game in game-projects/*/; do
        game_name=$(basename "$game")
        # Skip non-game directories
        [[ "$game_name" =~ ^(logs|2023|2026)$ ]] && continue
        
        # Grep for includes of this subsystem
        if grep -r "include.*$subsys_name" "$game" 2>/dev/null | grep -q "\.asm"; then
            used_in="$used_in, $game_name"
        fi
    done
    used_in="${used_in#, }"  # Remove leading comma
    
    echo "**Used in:** $used_in" >> "$OUTPUT_FILE"
```

Create `templates/new-game-template/README.md`:

```markdown
# New Game Template

Minimal scaffold for Thomson TO8 games using the engine.

## Quick Start

1. Copy this folder to `game-projects/my-game/`
2. Edit `config-linux.properties`: change `game.name` and `game.output`
3. Replace placeholder objects in `src/objects/` with your game objects
4. Build: `togen ./config-linux.properties`
5. Run: `to8 dist/my-game.fd`

## Structure

- `src/game-modes/` — Game screens (splash, main gameplay, menus)
- `src/objects/` — Game objects (player, enemies, projectiles, items)
- `src/global/` — Shared constants, macros, variables
- `config/` — Sprite sheets, tile maps, music (edit for your game)
- `dist/` — Build output (floppy disk, cartridge, SD card images)

## Subsystems Used

This template uses these engine subsystems (see `doc/engine-subsystems.md`):
- **graphics** — Render sprites and tiles
- **joypad** — Read player input
- **object-management** — Spawn, update, destroy game objects
- **collision** — Detect hitbox collisions
- **sound** — Play music and sound effects
- **level-management** — Load and manage levels

## Customization

1. **Replace player object** — Edit `src/objects/player/`
   - `player-sprite.asm` — Player sprite data
   - `player-logic.asm` — Player input and movement logic
   - Reference: `doc/examples/joypad/input-polling.asm`

2. **Add enemy objects** — Create `src/objects/enemy/`
   - Create new .asm files for each enemy type
   - Call `object-management:spawn-object` to create instances
   - Reference: `doc/examples/object-management/spawning.asm`

3. **Add collision** — Edit object logic files
   - Call `collision:check-rect-collision` each frame
   - Reference: `doc/examples/collision/collision-detection.asm`

4. **Customize graphics** — Edit `config/sprites.png`, `config/tiles.png`
   - Java generator converts to engine format during build

## Reference

Full engine documentation: `doc/engine-subsystems.md`
Code examples: `doc/examples/`
Example games: `game-projects/r-type/`, `game-projects/test/`
```

; From: r-type/src/main/main-loop.asm (lines 88-102)
; Example: How to read joystick input each frame
;
; Usage:
;   1. Call joypad:poll-input to read current input state
;   2. Check input flags (A register) for button state
;   3. Call player-logic functions based on input
;
; Dependencies: joypad subsystem

; Read joystick state and update player
; This runs each frame in the main game loop
poll-player-input:
    jsr joypad:poll-input    ; Read joystick (returns A = input flags)
    
    ; Check for up/down/left/right
    bit #$10             ; Bit 4 = joystick up?
    beq not-up
    jsr player:move-up
not-up:
    bit #$20             ; Bit 5 = joystick down?
    beq not-down
    jsr player:move-down
not-down:
    
    ; Check for fire button
    bit #$40             ; Bit 6 = fire button?
    beq not-fire
    jsr player:fire-weapon
not-fire:
    
    rts

; See full context: game-projects/r-type/src/main/main-loop.asm (lines 88-102)
